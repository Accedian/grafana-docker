Disk Usage Alert Guide

Troubleshooting high disk utilization

Affected alert

Disk usage

What does this alert mean?

The disk usage alert fires when the filesystem utilization on one or more monitored volumes exceeds the configured threshold. This is measured using standard host-level metrics (e.g., node_filesystem_avail_bytes) collected by Prometheus.

Impact

Running out of disk space can cause cascading failures across the platform:

  • Ingestion disruption — write operations fail when filesystems are full, stalling data pipelines.
  • Backup failures — scheduled backups cannot complete if target volumes lack space.
  • Query degradation — analytics services may become unresponsive if their working directories are exhausted.
  • Log loss — application and system logs stop being written, hindering further troubleshooting.

Investigation steps

  1. 1 Open the alert in Grafana and note the instance and mountpoint labels to identify which host and filesystem are affected.
  2. 2 Log into the affected host (or use your management console) and run df -h to confirm current utilization across all mount points.
  3. 3 Identify the largest consumers with du -sh /* (adjust the path to the affected mount). Look for rapidly growing directories such as logs, temporary files, or data staging areas.
  4. 4 Check whether any log rotation or cleanup cron jobs have stopped running or are misconfigured.

Remediation

Immediate relief

  • Rotate or compress historical log files.
  • Remove temporary files, old package caches, or orphaned data that is no longer needed.
  • If a specific application's data directory is the culprit, check whether it has its own cleanup or retention policy that can be adjusted.

Long-term capacity

  • Expand the filesystem or attach additional storage if usage reflects steady organic growth.
  • Review data retention policies and ensure automated cleanup is in place for all significant write paths.
  • Set up trending alerts (rate-of-change) to catch rapid growth before it reaches critical levels.

Verification

After freeing space or expanding the volume:

  • Run df -h to confirm utilization is below the alert threshold.
  • Wait for the next Grafana alert evaluation cycle and verify the alert state returns to Normal.
