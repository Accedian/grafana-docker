Druid & Query Engine Alert Guide

Troubleshooting the analytics query engine, data availability, and task scheduling

Affected alerts

Druid Status Check Druid Overlord Cluster View DataSource Availability All Druid Task Slots Full Historical Usage Metrics Service Reachability Query Node Count

What do these alerts mean?

The analytics platform uses a distributed query engine to serve dashboards and real-time queries. These alerts cover different failure modes:

  • Status / availability — the query engine or its data sources are not fully reachable. Some or all historical data may be unavailable for queries.
  • Cluster view mismatch — the cluster coordinator's view of connected nodes disagrees with what nodes report. This indicates a node connectivity or registration problem.
  • Task slots full — all ingestion task slots are occupied, meaning new ingestion tasks cannot start and will queue. Data freshness will degrade until slots free up.
  • Historical usage high — storage utilization on query-serving nodes is approaching capacity, which can cause eviction of data segments or query failures.
  • Service reachability / query nodes — the metrics query service cannot reach query-serving nodes, or the number of available query nodes has dropped below the minimum.

Impact

  • Dashboards may show gaps or fail to load if data segments are unavailable.
  • Queries may time out or return partial results.
  • New data ingestion stalls when all task slots are occupied.
  • High storage utilization on query nodes may trigger automatic segment eviction, causing additional query failures.

Investigation steps

  1. 1 Open the alert in Grafana and review the specific metric to determine which component is affected (query nodes, coordinator, task scheduler).
  2. 2 Check the query engine service logs on the affected nodes for errors related to connectivity, out-of-memory conditions, or disk space.
  3. 3 Verify disk utilization on query-serving nodes. Historical data segments are cached on local disk — if the disk fills up, the node cannot load or serve segments.
  4. 4 For task slot alerts, check whether long-running or stuck ingestion tasks are occupying all available slots. Identify and address the root cause of any stuck tasks.
  5. 5 For cluster view mismatches, check network connectivity between the coordinator and the affected nodes. Nodes may need to be restarted to re-register with the cluster.

Remediation

Data availability issues

Verify that all query-serving nodes are running and have sufficient disk space. If nodes have gone offline, restart them and monitor segment loading. The coordinator will automatically rebalance segments across available nodes.

Task slots full

Identify any stuck or long-running tasks and terminate them if appropriate. If the workload has legitimately grown, consider adding more task capacity. Review task scheduling to spread ingestion load more evenly.

High historical usage

Free disk space on query-serving nodes by removing stale data, or expand storage capacity. Review data retention rules to ensure old segments are being dropped on schedule.

Cluster view mismatch

Restart the affected node to force it to re-register with the coordinator. If the problem persists, investigate network connectivity between the node and the coordinator service.

Verification

  • Confirm all query-serving nodes are online and data sources show 100% segment availability.
  • Verify task slots have available capacity for new ingestion tasks.
  • Check that dashboard queries return complete data.
  • Wait for the next Grafana evaluation cycle and confirm alerts return to Normal.
