Airflow Ingestion Alert Guide

Troubleshooting batch data-ingestion alerts

Affected alerts

Airflow DAG Failed Airflow DAG Failed – custom reports Airflow DAG Failed – DHL Jobs Airflow Health Status Airflow job duration Airflow Jobs Stopped Airflow Batch DAG Failed

What does this alert mean?

These alerts monitor Apache Airflow DAG execution within the analytics platform. They fire when one or more of the following conditions is detected:

  • A DAG run has failed repeatedly within its evaluation window.
  • The Airflow scheduler is not reporting a healthy state.
  • No Airflow jobs have completed recently (jobs stopped).
  • A DAG's runtime has exceeded the expected duration threshold.

Impact

Data ingestion tasks are not completing on schedule. Batch processing jobs that aggregate, transform, or load data into the analytics platform are delayed. Dashboards and reports that depend on this data may show stale or incomplete results until the issue is resolved.

Investigation steps

  1. 1 Open the firing alert in Grafana and review the Labels section to identify the specific DAG ID and owner. The alert's evaluation history shows when the condition first triggered.
  2. 2 Navigate to the Airflow UI and locate the failed DAG. Inspect the most recent DAG run and its individual task logs to identify the root cause (e.g., dependency timeout, resource exhaustion, upstream data unavailability).
  3. 3 Check the Airflow scheduler health page. A healthy scheduler should show recent heartbeats and an active process. If the scheduler is down, other DAGs will also stop executing.
  4. 4 Confirm that upstream data sources are delivering records. If ingestion depends on external feeds, verify connectivity and recent data arrival timestamps.

Remediation

Transient DAG failure

If the root cause is resolved (e.g., a temporary network issue), clear the failed task instances and trigger a re-run from the Airflow UI.

Scheduler unhealthy

Restart the Airflow scheduler service. Monitor the heartbeat metric to confirm it recovers. If the scheduler continues to fail, review its resource allocation and logs for errors.

Long-running DAGs

If a DAG consistently exceeds its expected runtime, review task concurrency settings and consider breaking large tasks into smaller units. Adjust the alert's duration threshold if the longer runtime is expected (e.g., during backfill operations).

Verification

After remediation, confirm that:

  • The alert returns to the Normal state in Grafana after its next evaluation cycle.
  • New DAG runs complete successfully in the Airflow UI.
  • Downstream dashboards reflect current data.
